# 手動QAチェックリスト — GUIスケールとコンテナ画面レイアウト

OmniChest のコンテナ画面（チェスト/インベントリにオーバーレイする UI）は、GUIスケールとウィンドウ
サイズによって論理キャンバスが変化するため、描画自体の自動テストが難しい。以下を手動で確認する。

> 背景: 高DPI/4Kモニタで GUI Scale = Auto/8 のとき論理キャンバスが UI の必要サイズを下回り、
> レイアウトが崩れる不具合があった（修正済み）。再発検出のため、特に **8 / 9 / Auto** を重点確認する。
> 収まり判定ロジックの純粋関数版は `common` の `GuiScaleFit` にあり、`GuiScaleFitTest` で自動テスト済み。

## 確認マトリクス

各セルで下記5項目を確認する：

1. **崩れ無し** — グリッド/パネルが破綻していない
2. **はみ出し無し** — 全要素が画面内に収まる（左操作ヘルプの下端、右パネル、上部行、検索欄）
3. **オーバーレイ重なり無し** — 右パネルが中央グリッドに重ならない、上部行が互いに重ならない
4. **アイテム/スロット整合** — アイテムアイコンがスロットのマス目と一致（=正方形・ズレ無し）
5. **閉じた後の復元** — チェストを閉じたら HUD/ワールドがユーザー設定の GUI スケールへ戻る

## 追加の動作確認（実効スケールクランプ特有）

- ☐ スケール 8 / Auto でチェストを開いた瞬間に、UI が「収まるスケール」へ一段下がる（小さくなる）こと。
- ☐ その状態で **スロットのクリック / Shiftクリック / ドラッグ / クイックムーブ / ツールチップ** が
  正常（マウス座標のズレが無い）こと。
- ☐ チェストを **Esc で閉じる / 別GUIへ遷移 / ワールドへ戻る** いずれでも、HUD やワールドのGUIスケールが
  ユーザー設定値へ戻ること（復元漏れが無い）。
- ☐ チェストを開いたまま **ウィンドウサイズを変更** しても破綻せず追従すること。
- ☐ 他のModのコンテナ画面 / バニラのインベントリ・作業台などで、スケールがクランプ**されない**こと
  （= OmniChest対応コンテナ以外に波及しない）。
- ☐ 低スケール(1〜7)で、修正前と見た目が変わっていないこと（回帰）。

## 再現環境メモ

- 不具合再現環境: 高DPI(約3840×2160)モニタ + GUI Scale = Auto または 8。
- 期待挙動: 対応コンテナを開いている間だけ実効スケールが 7 相当（=収まる最大）へクランプされ、
  低スケールと同じ余裕あるレイアウトになる。閉じると即復元。
